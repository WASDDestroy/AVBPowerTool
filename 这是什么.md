# AVBPowerTool 说明书

> 这不是自述文档。

一个命令生成器，通过配置化的方式存储和生成 `avbtool.py` 可以识读的命令。尤其针对联想出品的使用 `AOSP 测试公钥` 来签名受保护镜像的设备设计。

## 什么是 AVB ？PowerTool 又是什么？

AVB 是 Android 可信启动的英文简写，用来保护高阶操作系统（Android）。正常情况下，通过使用私有的密钥对并保持 Bootloader 锁定，此系统可以有效防止当代技术手段篡改镜像。

除非......有厂商用了测试公钥。就是 Google 存放在 AVB 技术文档的 [Git 仓库](https://android.googlesource.com/platform/external/avb/+/main/)的 `external` 文件夹的那批东西。原本是用来测试的公开玩意，现在被直接拿来用在生产环境，有点恐怖的（哎，我说，`com.vvb2060.keyattestation` 应该添加检测 AOSP 测试公钥的功能）。

所以这个项目的目的就是让利用这个漏洞的变成愉快的交互式体验，方便充分利用这个漏洞来免解锁刷 Root 和制作 ROM！（当然，需要提前有一份配置文件）That's Power ~~I AM THE STORM THAT IS APPROACHING~~

## 如何使用？


### 1. 安装 Python

你需要一个 Python 运行时。

这个项目已经在 Termux (armv8, Android) 、x86-64 Ubuntu 22.04 LTS 、Windows 10 和 11 上测试过，这些平台的 Python 环境一般都是可以正常使用的。开发环境最开始是 Python 3.9 ，后续升级到了 Python 3.14 ，并在 Python 3.13 的设备上进行了部分开发和较高强度的使用，因此 `Python 3.9+` 暂定为本项目的 Python 需求。

对于 Windows ，请访问 [](https://www.python.org/downloads/windows/) 下载一个 Python 3.13 （截止本文完成时它是比较“主流”的 Python 版本）安装包。

对于 Linux ，用这个指令：

```shell
sudo apt-get install python3
```

对于 Termux ，请这样设置环境：

```shell
pkg install python3
```

如果你需要在其他终端模拟器，尤其是不带包管理器的终端模拟器，例如 *MT 管理器* ，你可能需要自己设置一个合适的 Python 环境。

### 2. 启动交互式界面

`main.py` 是本程序的入口点，如果使用下面的命令（Termux and common Linux distributions），不传递任何命令行参数，将直接启动交互式界面：

```shell
python3 main.py
```

对于 Windows ，一般需要使用下面两行命令中的**任意一行**：

```powershell
py main.py # Python launcher for Windows
python main.py # 有些时候 Python 启动器会出问题，这个时候这条命令可能很有用（如果系统安装有 Python 2 的话请谨慎使用）
```

关于交互式界面的说明请看 [交互式页面结构](./这是什么.md#交互式界面) 。

如果传递命令行参数，可以使用命令行模式调用本程序，不妨从 `python3 main.py --help` 开始。

## 交互式界面

交互式界面的结构如下，本地化功能会在不知道何时才能加上了，下面的目录树会提供大致的中文释义：

``` text
|-- View Current Config Info （查看当前活动配置的信息，这个页面具备本地化功能）
|   |-- 直接展示一个包含当前活动的配置文件全部信息的界面，按回车键可以返回主界面
|
|-- Read Image Info （读取镜像目录 `./Images` 中的部分或者全部镜像信息）
|   |-- 确认之后可以使用[选择器]选取镜像，关于选择器的用法请参考下一节
|
|-- Config Manager （简单的配置管理器）
|   |-- Activate a config （用配置库中的一个配置**覆盖**现有活动配置）
|   |-- Save current config to persistent storage （将当前活动配置存储到配置库，需要提供一个配置名）
|   |-- Help （显示创建配置文件所需的基本步骤）
|   |-- Import Config （导入由本工具创建的配置压缩文件）# 目前只支持识别工具根目录下的压缩文件
|   |-- Export Config （将本工具配置库中的配置导出为压缩文件）# 批量导出时可以导出为两种归档，每个配置文件一个归档或者若干个归档再整合到一整个主归档中
|   |-- Config Library Manager （删改配置库中的配置）
|       |-- Rename （选定配置后重命名此配置，需保证不重名，否则会提示错误）
|       |-- Delete （永久删除本配置）
|       |-- Activate （激活本配置，作用和 Config Manager 下的 "Activate a config" 是一致的）
|
|-- Sign Images （根据当前的配置文件签名镜像，如果配置了 `vbmeta` 信息则可以生成 vbmeta 镜像） # 请同样参考选择器一节
|
|-- Exit （退出交互模式，按 Ctrl + C 可以强制终止本程序）
```

## 选择器

选择器被用来选择文件和 UI 选项，可以使用方向 `↑` 和 `↓` 键或者 `W` 和 `S` 键来移动选项。

选择器支持两个模式：单选和多选，单选模式下按键提示中没有按键 `A` 的用法提示，只需要选择合适的选项然后按下 `回车` ；多选模式下则需要使用空格选定需要的选项，全部选择完成后再按回车确认。多选模式可以直接按 `A` 键全选或者取消全选所有条目。

在支持取消的界面会出现 `Esc` 按键的使用提示，按下 `Esc` 后选择器将返回一个空列表，从而取消动作。

## 命令行模式

使用命令行模式需要传递参数，下面是可用的参数列表：

```text
usage: AVBPowerTool [-h] {sign,read,save,set_active,import,export,about} ...

Image signing and configuration tool

positional arguments:
  {sign,read,save,set_active,import,export,about}
                        Available commands
    sign                Sign given images
    read                Read vbmeta info of given images
    save                Save current config to persistent storage
    set_active          Set a config as active
    import              Import config from zip archive
    export              Export config to zip archive
    about               Show about message

options:
  -h, --help            show this help message and exit
```

用法示例，签名 LKM 镜像组：

```shell
# Do something...
mv kernelsu_patched.img init_boot.img # 比如说已经获得了 KernelSU 修补的 init_boot ，这里为了让工具识别需要修改名称为分区名
python3 main.py import --file my_config.zip # 导入自定义配置，如果有配置的话不需要导入
python3 main.py set_active --name my_config # 激活此配置，工具必须有一个激活的配置才能工作
python3 main.py sign --images init_boot vbmeta # 根据配置文件签名镜像
# Lenovo Legion Y700 Gen2 to Gen4, Xiaoxin Pad Pro GT(Lenovo YOGA Tab), YOGA Tab 2025
# Do something else...
```

## 跨平台兼容

工具本身是跨平台的，并且已经经过测试。

原版 avbtool.py 面临两个问题：
1. 使用了一些 Linux 特有的设备，例如 `/dev/urandom` 来产生随机数
2. `fec` 难以在 Windows 上引入

第一点比较好解决，使用 Python 的 secrets 库即可。针对第二点，本工具引入了 Python 软件包 `reedsolo` 配合 `numpy` 实现的 FEC 算法，同样可以和 `avbtool` 以及 `libavb` 正常协作。

## 编写配置文件

如果要从零开始建立一个配置文件，除了按照帮助信息提供的方式进行操作，直接访问 `Core` 目录下的 `currentConfigs` 和 `currentKeySet` 文件夹虽然不规范，但是比创建两个名字独特的文件夹更快些。

1. 在 `currentConfigs` 文件夹下创建一个名为 `imageList.txt` 的文件，**每一行**包含本配置应当读取的全部**分区名称**（因此，不要添加 `.img` 扩展名）。可以使用半角井号 `#` 创建注释行，空行（换行符、只有空格的一行）将被忽略。
2. 将所需要的 `.pem` 文件放置在 `currentKeySet` 文件夹中。
3. （可选）将该配置关联的镜像组放置在 `Images` 工作目录下，启动工具读取一次**全部**镜像信息。
4. 使用 Config Manager 将配置持久化并导出。

## 修改配置

计划在 2.0 版本的配置管理器中提供一个内置的交互式编辑器，以更直观的方式展现和修改配置信息。

当前版本如需要修改配置，请访问配置文件夹并直接改动 `imageInfo.json` （和 `imageList.txt` ）。例如合入一个分区的信息、修改回滚指数、改动 `boot` 安全补丁等级、手动匹配公钥文件等。

对于高要求的配置编辑，建议配合文本比较工具完成操作。
